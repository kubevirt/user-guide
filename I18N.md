# Internationalization (i18n) for the KubeVirt User Guide

We use the mkdocs-static-i18n plugin for multiple language support.

English content goes to `main` during the development cycle. After a release, a versioned branch (e.g., `v1.7`) is cut from `main`. That branch inherits the i18n infrastructure (plugin config, build script, empty language directories), and translation PRs target it to appear in the released docs.

## Overview

We use the [mkdocs-static-i18n](https://github.com/ultrabug/mkdocs-static-i18n) plugin with the `folder` docs structure. English is the default language, with Spanish (es) and Chinese (zh) available. Untranslated pages fall back to the English version.

## How It Works

The i18n plugin expects language subdirectories under `docs/` (e.g., `docs/en/`, `docs/es/`, `docs/zh/`). Since we keep content directly in `docs/` without language prefixes, we restructure at build time.

The `hack/internationalize.sh` script does the restructuring:

1. Creates `docs/en/` and moves all documentation content into it
2. Preserves shared assets (`assets/`, `stylesheets/`, `_redirects`) at the top level
3. Copies translation directories from `i18n/` into `docs/`

This script is used in two places:
- **Local development**: The `make run_i18n` target runs it inside the container
- **Netlify builds**: `netlify.toml` calls it with the `--netlify` flag to apply additional site URL and docs_dir transforms

## Directory Structure

```
user-guide/
├── docs/            # English source content (default)
├── i18n/
│   ├── es/          # Spanish translations
│   └── zh/          # Chinese translations
├── hack/
│   └── internationalize.sh  # Build-time restructuring script
└── mkdocs.yml       # Plugin configuration
```

## Adding a New Language

1. Create a new directory under `i18n/` (e.g., `i18n/fr/` for French)
2. Add a `.gitkeep` file to track the empty directory
3. Add the language to `mkdocs.yml` in both `plugins.i18n.languages` and `extra.alternate`
4. Update `hack/internationalize.sh` to clean up the new language directory (the `rm -rf` line)
5. Add translated markdown files to the new language directory, mirroring the structure under `docs/`

## Adding Translations

Place translated markdown files in the appropriate `i18n/<locale>/` directory, mirroring the path structure of `docs/`. For example, to translate `docs/cluster_admin/installation.md` to Spanish, create `i18n/es/cluster_admin/installation.md`.

Untranslated pages fall back to the English version.

## Local Development

To test with translation support locally:

```bash
make build_img       # Build the container image (first time only)
make run_i18n        # Run site with i18n at http://localhost:8000
make stop            # Stop the running site
```
